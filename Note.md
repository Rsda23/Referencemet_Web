# **SEO (Search Engine Optimization)**  
- **Effet durable** : Pas besoin de s'en occuper en continu.  
- **Coût** : Gratuit ou à moindre coût.  
- **Objectif** : Rendre le site ergonomique, fiable et accessible, avec une optimisation technique efficace.  

---

## **Quelques statistiques clés**  
- **Noms de domaine créés chaque année** : Environ 350 à 360 millions, en constante augmentation.  
- **WordPress** :  
  - 10 millions d'articles créés chaque jour.  
  - Représente 65 % des parts de marché mondiales (2/3 des sites web sont développés avec WordPress).  
- **Parts de marché des moteurs de recherche** :  
  - **Google** : 93 %.  
  - **Bing** : 2 %.  
- **Recherche Google** :  
  - 9 milliards de requêtes par jour.  
  - Requêtes composées de 1 à 4 mots en moyenne.  
  - 60 à 70 % du trafic web provient des mobiles.  
  - 25 % des recherches incluent la géolocalisation.  
  - Moins de 5 % des 2 milliards de sites web sont en français.  

---

## **SEA (Search Engine Advertising)**  
- **Effet** : Court terme (les résultats disparaissent si les paiements cessent).  
- **Avantage** : Retour sur investissement très favorable.  
- **Objectif** : Positionner un site web sur des mots-clés spécifiques.  

---

## **Référencement et positionnement**  
1. **Référencement** : Les pages web sont référencées lorsqu'elles sont incluses dans les bases de données des moteurs de recherche.  
2. **Positionnement** : Rang d'une page dans les résultats naturels (hors SEA).  

---

## **Définitions des termes clés**  
- **SEO** : Techniques visant à optimiser le positionnement des pages dans les résultats naturels des moteurs de recherche (nécessite que le site soit déjà référencé). Cela inclut :  
  - L'optimisation technique du site.  
  - La modification des contenus.  
  - L'indexation.  
  - Le suivi des résultats.  
- **AEO (Answer Engine Optimization)** : Techniques pour optimiser le contenu afin de répondre aux questions des utilisateurs et d’apparaître en "position 0" dans les moteurs de recherche.  
- **SMO (Social Media Optimization)** : Techniques pour accroître la visibilité des sites via les réseaux sociaux (Facebook, Instagram, etc.).  
- **SEM (Search Engine Marketing)** : Ensemble des techniques de marketing en ligne (inclut SEO, SEA et SMO).  
- **SERP (Search Engine Results Pages)** : Pages de résultats des moteurs de recherche, comportant :  
  - Les liens organiques (résultats naturels, SEO).  
  - Les liens sponsorisés (résultats payants, SEA).  

---

## **Backlink et Netlinking**  
- **Backlink** : Lien pointant vers un site web depuis une autre page web (interne ou externe). Impacte la popularité et le référencement.  
- **Netlinking** : Stratégie de création de backlinks pour améliorer le référencement d'un site.  

---

## **Syntaxes de recherche Google**  
- Google utilise un opérateur "ET" implicite pour plusieurs mots-clés.  
- Google effectue de la **lemmatisation** (analyse des variantes des mots) et ignore :  
  - Les fautes mineures.  
  - La casse.  
  - Les accents.  
- Quelques syntaxes utiles :  
  - `..` : Recherche entre deux valeurs (ex. : "prix 100..200").  
  - `-` : Exclut un terme (ex. : "chocolat -blanc").  
  - `""` : Recherche exacte (ex. : "meilleur restaurant Paris").  
  - `filetype:` : Recherche un format spécifique (ex. : "rapport filetype:pdf").  
  - `site:` : Recherche limitée à un domaine (ex. : "site:wikipedia.org").  
  - `link:` : Affiche les backlinks d'une page.  
  - Autres opérateurs : `cache:`, `info:`, `related:`, `allintitle:`, `allinurl:`.  

---

## **Autres concepts importants**  
- **Liens naturels** : Liens pointant naturellement vers un site web (non sponsorisés).  
- **Champs de recherche** : Les moteurs analysent les titres, URL et descriptions pour les résultats.  
- **Moteur de réponse** : Google anticipe les besoins des utilisateurs en fournissant des réponses directes.  
- **IA dans les moteurs de recherche** : Basée sur un fonctionnement inspiré du cerveau humain.  
- **SGE (Search Generative Experience)** : Contenu généré par IA en lien avec les recherches.  
- **Knowledge Graph** : Fournit des contenus enrichis liés à la recherche initiale.  
- **One Box** : Zone de mise en avant des éléments clés pour l'utilisateur.  
- **Feature Snippet** : Extrait mis en avant pour répondre directement à des questions comme "comment", "combien", "qu'est-ce que".

## **Techniques de référencement**  
- **White Hat SEO** : Référencement respectant les consignes officielles.  
- **Black Hat SEO** : Techniques contraires aux consignes pour manipuler le référencement.

## **Étude de marché**

- Service utilisé par les entreprises pour **gagner en productivité**.

## **Importance des mots-clés**  
- Les mots-clés sont la **source d’un bon référencement** :  
  - **Pour les moteurs de recherche** : ils permettent d’indexer correctement les pages.  
  - **Pour les utilisateurs** : ils aident à trouver les pages pertinentes.  

---

## **Composition d’une page web optimisée**  
1. **Dans l’entête de la page (HEAD)** :  
   - Invisible pour l’utilisateur, mais lue par les moteurs de recherche et les navigateurs.  
   - Contient des informations essentielles comme :  
     - Le titre.  
     - La description.  
     - Les mots-clés.  

2. **Dans le corps de la page (BODY)** :  
   - Partie visible de la page où se trouve le contenu principal.  

3. **Dans les adresses (URL)** :  
   - Les mots-clés dans l’URL sont très importants pour le référencement.  

---

## **Balise Title**  
- Le contenu de la balise `title` :  
  - S’affiche dans les SERPs (Search Engine Results Pages).  
  - Apparaît sur l’onglet de la page dans le navigateur.  
- **Objectif** :  
  - Contenir des mots-clés pertinents.  
  - Informer clairement l’utilisateur sur le contenu de la page.  

---

## **Conseils de rédaction pour la balise Title**  
1. **Clarté et précision** :  
   - Se limiter à une dizaine de mots maximum (moins si possible).  
2. **Optimisation des mots-clés** :  
   - Trouver une expression qui inclut le plus de mots-clés possible.  
   - Placer les mots-clés en début de phrase.  
3. **Personnalisation** :  
   - Donner un titre unique à chaque page.  
4. **Éviter** :  
   - Les mots-clés redondants.  
   - Les termes trop génériques.  
5. **Caractères** :  
   - Le nombre de caractères reconnus varie selon les moteurs de recherche.  

## **Balise Meta**  
- La **balise meta** est une balise HTML qui permet de définir des **métadonnées** qui ne sont pas affichées à l’écran.  
- Problème courant : Trop de pages n’en contiennent pas encore.  
- **Évolution des moteurs de recherche** :  
  - Ces balises sont devenues **moins importantes pour le référencement**.  
  - Elles restent cependant **utiles pour d’autres raisons** (ex. : informations sur le contenu).  
- **Bonne pratique** :  
  - Inclure des balises meta sur **toutes les pages**.  
  - Les adapter en fonction du **contenu spécifique** de chaque page.  

---

## **Balise Meta Description**  
- La **balise meta description** est un résumé de la page.  
- **Affichage** :  
  - Elle apparaît dans les SERPs (Search Engine Results Pages).  
- **Objectifs** :  
  1. Être **claire, concise et informative**.  
  2. Contenir des **mots-clés pertinents**.  
  3. **Inciter l’utilisateur à cliquer** sur le lien.  

---

## **Balise Meta Robots**  

- La **balise meta robots** est utilisée pour indiquer aux robots des moteurs de recherche comment ils doivent se comporter avec une page.  
- **Fonctionnalités principales** :  
  - Définir si la page doit être **indexée** ou non.  
  - Préciser si les robots doivent **suivre les liens** présents sur la page ou non.

  ```html
  <meta name="robots" content="noindex, nofollow">
  ```

## **Balise Meta Robots : Directives importantes**  

- **`nosnippet`** : Empêche l'affichage d'un extrait de la page dans les SERPs.  
- **`noarchive`** : Empêche l’archivage de la page.  
- **`unavailable_after`** : Indique la date après laquelle la page ne doit plus être indexée.

- Fréquence d’apparition d’un contenu spécifique sur un site web.

## **Spamdexing**  
- Technique consistant à **répéter un mot-clé** pour améliorer le référencement.  
- **Ne fonctionne plus** :  
  - Peut entraîner un **blacklistage** ou une **sanction de type sandbox**.  

### **Blacklist vs Sandbox**  
- **Blacklist** : Liste rouge des sites bannis.  
- **Sandbox** : Environnement où le site perd en visibilité et son indexation devient chaotique.  

---

## **Optimisation SEO et structure des liens**  

- **Durée d’optimisation** :  
  - Optimiser une page pour **plus de trois mots-clés** est complexe.  

- **Composition des textes de liens** :  
  - Éviter les liens **génériques** (ex. : "cliquez ici").  
  - Privilégier des liens **précis et explicites**.  
  - Assurer l’interconnexion de toutes les pages du site pour faciliter la **cartographie par les robots Google**.  

---

## **Hiérarchie et structuration du contenu**  

- **Balises titres et sous-titres (`h1`, `h2`, `h3`...)** :  
  - Bien les mettre en valeur pour structurer le contenu.  
  - Les moteurs de recherche y accordent une importance primordiale.  
  - **Respecter l’ordre logique** des titres pour une meilleure compréhension.  

- **Plan du site (`sitemap.xml`)** :  
  - Peut être un **plus** pour le référencement.  
  - Les robots apprécient cette structure.  

---

## **Utilisation de l’IA dans le SEO**  

- **À utiliser pour** :  
  - Analyse de données.  
  - Analyse des tendances et des mots-clés populaires.  
  - Étude de la concurrence.  

- **À éviter pour** :  
  - La **partie technique** du site.  
  - La création automatique de contenu.  
  - L’optimisation structurelle (mieux vaut vérifier manuellement à la fin du projet).  

---

## **Optimisation des médias (images, vidéos, sons)**  

### **Images : Bonnes pratiques**  

**Soigner le nom des fichiers** (inclure des mots-clés).  
**Utiliser des tirets** pour séparer les mots (éviter les espaces ou underscores).  
**Formats d’image recommandés** :  
  - **JPEG (`.jpg`)** : Mode **RBG**, idéal pour les photographies.  
  - **PNG (`.png`)** : Supporte la **transparence** (mode RGBA).  
  - **SVG (`.svg`)** : **Vectoriel**, ne perd pas en qualité, même compressé.  
  - **WebP (`.webp`)** : Format **Google**, efficace pour la compression, entre SVG et JPEG.  
**Remplir l’attribut `alt` des images** avec des mots-clés liés au contenu.  

### **Vidéos : Bonnes pratiques**  

**Éviter AJAX, jQuery et Flash**, car mal interprétés par les moteurs de recherche.

## **Génération de trafic et SEO**  
- **Tout élément générant du trafic** booste le référencement **s’il est bien utilisé**.  

---

## **Référencement dans Google Actualités**  

Avoir une section **identifiable** dédiée aux actualités.  
**Respecter les règles** :  
  - Une **URL structurée** avec des **mots-clés et des tirets**.  
  - Une **date récente** et visible.  
  - Une **publication régulière** d’articles.  
  - Un **dossier spécifique** pour les contenus d’actualité.  
  Utiliser les outils proposés par Google pour comprendre son fonctionnement.  

---

## **Contenu généré par les utilisateurs : Attention !**  

**Ne jamais indexer le contenu généré par les internautes** (ex. : commentaires, forums...).  
- Peut contenir des **liens frauduleux** ou du **spam**, nuisant au référencement du site.

## **Indice de confiance**  

- Facteur influençant le **positionnement d’un site dans les résultats de recherche**.  
- Google attribue un **niveau de confiance** en fonction de plusieurs critères :  
  - **Taille du site** : Un site bien structuré avec un grand nombre de pages est mieux perçu.  
  - **Ancienneté** : Plus un site est ancien, plus il est considéré comme fiable.  
  - **Trafic** : Un fort trafic est un indicateur de pertinence et de popularité.  
  - **Niveau de véracité des informations** : La qualité et la crédibilité du contenu impactent la confiance accordée par Google.  

- **Conséquence pour les nouveaux sites** :  
  - Ils sont généralement **moins bien considérés** par Google au départ.  
  - Il faut du temps et du contenu de qualité pour gagner en crédibilité et améliorer son positionnement.  

# Certificat SSL  
Le **certificat SSL** est un protocole de sécurité indispensable pour le web. Il garantit la **confidentialité** et l’**intégrité** des données échangées entre un navigateur web et un serveur en les **chiffrant**.  

Sans **HTTPS**, Google peut flagger le site comme non sécurisé. De plus, les certificats SSL ont une durée de validité limitée et doivent être renouvelés régulièrement.  

# Sécurité des données  
- **Confidentialité** : assurer que les données sont **chiffrées** et ne peuvent pas être lues par un tiers.  
- **Intégrité** : vérifier que le message n’a pas été **modifié** (via un **checksum**).  

# URL  
L’**URL** (*Uniform Resource Locator*) est l’adresse d’une ressource sur le web. En général, cette ressource est un document **HTML**.  

# Sitemap  
Un **sitemap** au format **XML** permet d’indiquer à Google la structure du site en listant uniquement les pages à indexer. Cela **facilite le référencement**.  
C’est la méthode la plus sûre pour informer Google des pages du site. On peut soumettre la **sitemap** via **Google Search Console**.  

# Référencement et technologies web  
- **React** et les frameworks JavaScript sont peu optimisés pour le référencement, mais offrent une excellente expérience utilisateur.  
- Il faut éviter :  
  - Les balises vides  
  - Les erreurs **404**  
  - Les redirections inutiles  
- Lors d’un changement de domaine, une **redirection 301** doit être mise en place pour transférer le référencement sans générer d’erreurs **404**.  

# robots.txt  
Le fichier **robots.txt** informe les moteurs de recherche sur les pages à indexer ou à ignorer.  
Sans ce fichier, **toutes** les pages du site peuvent être indexées par défaut.  

# Mobile et SEO  
Un site web moderne doit être **responsive** pour éviter de perdre en référencement. Google impose plusieurs critères :  
- Ne pas utiliser **Flash** sur mobile  
- Espacer suffisamment les liens pour faciliter les clics
- Éviter les publicités interstitielles qui occupent tout l’écran  

# Fiche établissement  
Une **fiche établissement** permet d’améliorer le **référencement local** d’une entreprise. Ajouter des données comme l’adresse et les horaires facilite son indexation par Google.
